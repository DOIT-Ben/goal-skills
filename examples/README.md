# Examples

These examples are public regression samples for `goal-skills`. They show when to compile one final goal, when to stop, and how to keep the current conversation ahead of stale project evidence.

| Example | Main goal type | What it protects |
| --- | --- | --- |
| [messy-project](messy-project.md) | Discover / Map | Do not edit before understanding a messy project |
| [broken-app](broken-app.md) | Repair / Restore | Restore a broken local baseline before expanding scope |
| [product-idea](product-idea.md) | Idea / Product Slice | Turn an idea into a minimum product slice, not full SaaS |
| [direct-repair](direct-repair.md) | Repair / Restore | Direct final goal for local repair and health validation |
| [research-spec](research-spec.md) | Research / Spec | Produce evidence-backed spec before implementation |
| [create-from-zero](create-from-zero.md) | Create From Zero | Build the first local usable artifact from notes |
| [add-extend](add-extend.md) | Add / Extend | Add one capability without broad refactor |
| [refactor-reorganize](refactor-reorganize.md) | Refactor / Reorganize | Preserve behavior while improving structure |
| [polish-improve](polish-improve.md) | Polish / Improve | Improve user-facing docs without code changes |
| [automate-dry-run](automate-dry-run.md) | Automate | Automate repeated work with dry run and logs |
| [experiment-evaluate](experiment-evaluate.md) | Experiment / Evaluate | Compare options with metrics before choosing |
| [release-publish](release-publish.md) | Release / Publish | Publish only with explicit authorization and visibility checks |
| [high-risk-external-write](high-risk-external-write.md) | Repair / Restore | Stop before production deploys or database writes |
